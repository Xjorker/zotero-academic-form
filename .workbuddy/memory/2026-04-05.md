# 2026-04-05

## CV 结构化解析功能（新接口）

### 问题
结构化表（academic_form_person/projects/patents）全空，因为前端上传 CV 时只调了 `/kb/import/document`（存文本分块），从未调 `/kb/import/author` 等接口。缺少 CV → 结构化数据 的提取环节。

### 方案
新增 `/kb/import/parse_cv` 接口，用 LLM 从 CV 文本提取结构化数据自动导入知识库。

### 改动文件
- `server/word_fill_app/api/kb_routes.py`：新增 `/kb/import/parse_cv` 端点
  - 支持两种输入：content（纯文本）或 file_base64（PDF/DOCX，后端自动提取）
  - 复用 `agent_routes._create_llm()` 创建 LLM 实例
  - `_extract_structured_from_cv()` 函数：system prompt 让 LLM 输出 JSON（author/projects/patents/datasets）
  - 提取后自动调用 `import_author_info()` / `import_projects()` / `import_patents()` / `import_datasets()`
- `src/modules/preferenceScript.ts`：CV 上传成功后自动调用 parse_cv
  - 在 `/kb/import/document` 成功回调中，自动获取 LLM 配置并调用 `/kb/import/parse_cv`
  - 对二进制文件传 `file_base64 + source`，对纯文本传 `content`
  - LLM 未配置时跳过，解析失败不影响主流程（文档已成功导入）

### 用户流程变化
用户只需：上传 CV → 自动完成两步
1. `/kb/import/document`：文本分块 + 向量化（RAG 检索用）
2. `/kb/import/parse_cv`：LLM 提取结构化数据（SQL 查询用）

## 填表质量优化（3个问题修复）

### 问题诊断
1. **基本简况没填**：`parse_cv` 截断到 12000 字符（CV 有 15037），丢失了后半部分的项目/专利/经费章节；KB 缺少 position/join_date 字段
2. **语言不统一**：`academic_form_person` 存的是英文（"Wuhan University"），但模板是中文的，Agent 直接把英文填入中文模板
3. **研究生培养/创新团队没填**：Agent 只调了 1 次 RAG，且查询词不够精准；parse_docx 没有返回表格前的段落标题

### 修复1：parse_cv 截断 → kb_routes.py
- `content[:12000]` → `content[:30000]`，避免丢失 CV 后半部分（项目/专利通常在后半）

### 修复2：语言一致性 → graph.py + kb_routes.py
- graph.py SYSTEM_PROMPT 新增「语言一致性规则」：
  - Agent 检测文档表头语言，统一填充语言
  - 英文数据填入中文文档时自动翻译（"Wuhan University" → "武汉大学"）
  - 人名保留原文，描述性文字翻译
- kb_routes.py parse_cv 提示词也加了语言一致性要求

### 修复3：RAG 多主题检索 → graph.py + parse_docx.py
- graph.py SYSTEM_PROMPT 执行流程从4步扩展到5步：
  - 步骤4 明确要求根据不同表格主题做 1~3 次 RAG 检索
  - 给出具体查询词示例（教学/团队/工作经历/项目）
- parse_docx.py 重写为遍历 w:body 子元素，记录每个表格前的段落标题
  - 新增 `context` 字段（如 "教学情况 | 研究生培养情况"）
  - Agent 通过 context 判断每个空表格应该填什么内容
- RAG 部分说明大幅增强，列出 RAG 能找到的所有信息类型
