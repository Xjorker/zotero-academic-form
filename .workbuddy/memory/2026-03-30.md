## 架构设计讨论（下午）

### 新功能可行性分析：个人知识库 + Text2SQL 动态填表

**结论：完全可行**，讨论了以下几个要点：

1. **Schema 自动派生**：不需要用户手动维护 Schema 文件，每次填表前通过 `PRAGMA table_info` 实时从 SQLite 读取表结构，自动生成供 LLM 使用的 Schema 描述，永远与数据库同步。

2. **动态 SQL 执行**：`Zotero.DB.queryAsync()` 接收普通字符串，LLM 生成的动态 SQL 和写死的 SQL 在执行层面完全等价。安全防护：前端检查只允许 SELECT 语句。

3. **EAV 模式**：用 `kb_schema`（字段语义） + `kb_records`（entity_id/attr/value）两张表支持灵活自定义字段，现有固定表保留并存。

4. **Agent 化可行性**：Zotero 插件支持 Function Calling 模式的 Agent Loop，需新增 `agentRunner.ts` + `toolRegistry.ts`，`llmService.ts` 扩展 `chatWithTools()` 方法，现有模块包装成工具即可。

5. **架构图**：生成了两版架构图（architecture.html / architecture2.html），v2 采用左右双列布局，左列知识库构建路径，右列填表执行路径，中间汇合后走原有 Python 后端（零改动）。

**关键集成点**：`academicForm.ts` 的 `getRawAuthorInfo()` 之后做数据合并。
