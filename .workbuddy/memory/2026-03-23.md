# 2026-03-23 工作日志

## 主要工作

### ZIP 修复（server/word_fill_app/main.py）
- 添加了 `_repair_zip()` 函数，通过扫描 `PK\x03\x04` 本地文件头重建损坏的 ZIP 中心目录
- 修复 `fill_document_base64` 中的逻辑错误：原来"修复"后仍使用损坏的原始数据，现在修复后的 content 会被重新赋值使用

### 前端优化（src/modules/academicForm.ts）
- 统一 base64 编码：原先 `base64Content`（LLM 用）和 `docxBase64`（填表用）各自独立编码，实际是同一个 uint8，改为只编码一次 `docxBase64` 复用
- 修复文件保存错误：`writeByteArray` 在 XPCOM 中不接受 `Uint8Array`，必须用 `Array.from(uint8Array)` 转为纯 JS 数组

### 关键结论
- Zotero 7（Firefox XPCOM）中 `nsIBinaryOutputStream.writeByteArray` 只接受 `Array<number>`，不接受 `Uint8Array`
- 正确写法：`binaryStream.writeByteArray(Array.from(uint8Array), byteArray.length)`
