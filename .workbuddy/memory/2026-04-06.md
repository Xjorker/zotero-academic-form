# 2026-04-06 工作日志

## 分批填表论文数据丢失 Bug 修复

### 问题
用户反馈：调用 `fill_docx(target_tables="papers")` 填论文后，再调用 `fill_docx(target_tables="others")` 填其他表格时，论文数据丢失。

### 根因
`tools_wrapper` 在 `fill_docx` 执行后，**没有将返回的 `docx_base64` 更新到 state 中**。

执行流程：
1. 第一次 `fill_docx(papers)`：读取 `state.docx_base64`（原始文档）→ 填论文 → 返回新 docx_base64
2. **关键缺失**：返回的新 docx_base64 没有写回 `state["docx_base64"]`
3. 第二次 `fill_docx(others)`：`tools_wrapper` 仍注入原始 `state.docx_base64` → 论文数据被覆盖

### 修复
`server/word_fill_app/agent/graph.py` 的 `tools_wrapper` 中，在 `tool_node.invoke(state)` 之后：
1. 从返回的 `ToolMessage` 中提取 `fill_docx` 的结果
2. 如果 `success=True` 且包含 `docx_base64`，则：
   - 更新返回的 state dict：`result["docx_base64"] = new_docx_base64`
   - 同步更新模块级变量：`set_state_docx_base64(new_docx_base64)`
