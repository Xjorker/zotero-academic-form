## 新增功能：构建向量索引（2026-04-07 下午）✅ 已完成

### 需求
用户右键一个或多个有 PDF 的论文条目，选择"构建向量索引"，系统为 PDF 构建向量索引并导入到 Chroma 向量库。

### 实现内容

1. **注册右键菜单** `context-build-vector-index`（在 authorProfile.ts 中）

2. **新增函数**：
   - `registerVectorIndexMenu()` - 注册右键菜单
   - `buildVectorIndexForItems(items)` - 主流程：过滤有 PDF 的论文、逐个调用索引 API
   - `getItemPdfBase64(item)` - 获取论文的 PDF 附件并转为 Base64
   - `readFileAsBase64(filePath)` - 读取本地文件为 Base64
   - `importPdfToVectorIndex(item, pdfBase64, source)` - 调用后端 `/kb/import/document` API
   - `getServerUrl()` - 获取后端服务器 URL

3. **支持的条目类型**：journalArticle、conferencePaper、book、bookSection

4. **处理流程**：
   - 过滤出有 PDF 附件的条目
   - 逐个获取 PDF 并转为 Base64
   - 调用后端 `/kb/import/document` 端点（已有功能）
   - 后端自动：PDF 文本提取 → 分块 → Chroma 向量索引

5. **后端 API**：`POST /kb/import/document`
   - 请求体：`source`, `file_base64`, `doc_type`, `title`, `chunk_size`, `chunk_overlap`, `metadata`
   - 元数据包含：doi, year, authors, zotero_item_id

### 修改文件
- `src/modules/authorProfile.ts` - 添加向量索引相关函数 ✅
- `src/hooks.ts` - 导入并调用 `registerVectorIndexMenu()` ✅
