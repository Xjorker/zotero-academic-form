> 基于 LakeFill 的设计理念，针对学术成果填表 Agent 的具体改进建议
> 项目：Zotero 插件 · 学术成果填表系统

## 一、现状分析

### 1.1 当前 Pipeline

```
parse_docx → query_kb → hybrid_rag → fetch_academic → fill_docx
```

| 阶段 | 现状 | 缺失能力 |
|------|------|---------|
| 数据采集 | ✅ KB + DBLP/CrossRef API | ❌ 置信度评分 |
| 填充策略 | ✅ 硬校验层（危险字段清空） | ❌ 软校验 / 双模式 |
| 结果校验 | ✅ 论文截断检测 | ❌ 实体等价验证 |
| 片段排序 | ✅ RRF 混合检索 | ❌ 细粒度重排 |

### 1.2 当前三大痛点

1. **"填了不知道对不对"** — 缺少置信度评分，LLM 填完无法自我评估
2. **"严格 vs 宽松" 一刀切** — 所有字段用同一策略，危险字段可能误填，叙述性字段可能填不上
3. **"实体误匹配"** — 期刊名、论文名近似但不等于实际值（如"MLLM" ≠ "Multi-Modal LLM"）

## 三、阶段一：置信度评分层（高优先级，2-3天）

### 3.1 设计目标

让 LLM 填充时同时输出**置信度**，系统根据置信度决定：
- 直接填充（高置信度）
- 标记后填充（中置信度）
- 留空（低置信度）

### 3.2 改动位置

```
graph.py (system_prompt) → LLM 填充时同时输出置信度
```

## 四、阶段二：双模式填充 + 三维度评估（中优先级，3-5天）

### 4.1 设计目标

对不同类型字段采用不同填充策略：
- **Strict Mode**：危险字段（事实性），只信 KB/API 显式返回
- **Relaxed Mode**：叙述性字段，允许 LLM 推断，但需标注置信度

对检索回来的 CV 片段做三维度评估（借鉴 LakeFill 的 Checklist）。

```python
# hybrid_rag.py
RAG_EVALUATION_PROMPT = """
对检索回来的 CV 片段，评估三个维度：

1. Existence（存在性）：
   - 该片段是否包含当前表格字段需要的信息？
   - 是 → 继续评估；否 → 直接丢弃该片段

2. Relevance（相关性）：
   - Highly Relevant：该片段描述的是同一个人的经历/成果
   - Somewhat Relevant：该片段描述的是同领域但非本人的内容
   - Not Relevant：实体明显不匹配

3. Logical Consistency（逻辑一致性）：
   - Fully Consistent：时间线合理，无矛盾
   - Partially Consistent：细节有小矛盾，但整体可信
   - Not Consistent：时间/逻辑明显矛盾（如"2024年在读"vs"2019年已毕业"）

只选用三维度全通过的片段做填充。
"""
```

### 4.4 实现方案

```python
# hybrid_rag.py — 在检索后加一层评估
def hybrid_rag_with_evaluation(query: str, top_k: int = 10) -> list[dict]:
    # 1. 原始检索
    candidates = hybrid_retrieve(query, top_k=20)

    # 2. 三维度评估（用 LLM 批量评估，节省 token）
    evaluated = []
    for candidate in candidates:
        scores = evaluate_three_dimensions(query, candidate.content)
        if scores["existence"] == "Yes" and \
           scores["relevance"] in ["Highly Relevant", "Somewhat Relevant"] and \
           scores["consistency"] in ["Fully Consistent", "Partially Consistent"]:
            evaluated.append({**candidate, "dimension_scores": scores})

    # 3. 按 Relevance 排序，取 top-k
    evaluated.sort(key=lambda x: x["dimension_scores"]["relevance"], reverse=True)
    return evaluated[:top_k]
```

```python
# entity_verification.py
import json
from pathlib import Path

class EntityVerificationCache:
    def __init__(self, cache_path="entity_verification_cache.json"):
        self.cache_path = Path(cache_path)
        self.cache = self.load()

    def load(self):
        if self.cache_path.exists():
            return json.load(open(self.cache_path))
        return {}

    def save(self):
        json.dump(self.cache, open(self.cache_path, "w"), ensure_ascii=False)

    def verify(self, entity_a: str, entity_b: str) -> bool:
        """判断两个实体名是否指向同一实体"""
        key = tuple(sorted([entity_a, entity_b]))
        if key in self.cache:
            return self.cache[key]

        # 用 LLM 判断
        result = self.llm_judge(f"{entity_a} 和 {entity_b} 是否是同一实体？")
        self.cache[key] = result
        self.save()
        return result

    def verify_venue(self, venue_short: str, venue_full: str) -> bool:
        """验证期刊缩写和全称是否等价"""
        return self.verify(venue_short, venue_full)
```

## 七、置信度体系详细设计

### 7.1 置信度来源

| 来源 | 获取方式 | 精度 |
|------|---------|------|
| API logprobs | DBLP/CrossRef 返回 logprobs | 高 |
| 检索一致性 | 同一字段多次检索结果一致性 | 中 |
| LLM 自评 | LLM 在输出中附带 | 低（但可作为参考） |
| 长度归一化 | NLN 公式对 logprobs 几何平均 | 高 |

## 八、可复用的代码模块

### 8.1 置信度计算模块

```python
# confidence.py
import numpy as np
from typing import List

def length_normalized_confidence(log_probs: List[float]) -> float:
    """
    NLN: Length-Normalized Logprob Confidence
    几何平均 + 长度归一化
    """
    if not log_probs:
        return 0.0
    L = len(log_probs)
    # 数值稳定性：使用 log-sum-exp
    return np.exp(np.mean(log_probs))

def entropy_confidence(top_logprobs: List[List[float]]) -> float:
    """
    基于 token 分布熵的置信度
    低熵 = 模型很确定
    """
    entropies = []
    for probs in top_logprobs:
        probs = np.array(probs)
        probs = probs / probs.sum()  # 归一化
        e = -np.sum(probs * np.log(probs + 1e-10))
        entropies.append(e)
    avg_entropy = np.mean(entropies)
    return np.exp(-avg_entropy)

def combined_confidence(llm_conf: float, log_probs: List[float], top_logprobs: List[List[float]]) -> float:
    """
    综合三种置信度
    权重可调
    """
    nln = length_normalized_confidence(log_probs)
    ent = entropy_confidence(top_logprobs)
    # 加权几何平均
    combined = (llm_conf ** 0.3) * (nln ** 0.4) * (ent ** 0.3)
    return combined
```

## 九、总结

| 改进方向 | 解决的问题 | 优先级 | 工作量 |
|---------|----------|--------|--------|
| 置信度评分层 | 填了不知道对不对 | 🔴 高 | 中 |
| 实体验证缓存 | 期刊/论文名近似误匹配 | 🔴 高 | 小 |
| Strict/Relaxed 双模式 | 危险字段被误填 | 🟡 中 | 小 |
| 三维度片段评估 | 检索质量无法保证 | 🟡 中 | 中 |
| Reranker 层 | RRF 排序不够精细 | 🟢 低 | 大 |

**核心思路**：借鉴 LakeFill 的**"让 LLM 知道它什么时候该严格、什么时候可以放松"**这一设计理念，为填表 Agent 增加置信度感知能力，从"盲目填充"升级为"可信填充"。
